# Infrastructure (Terraform)

Provisions the full production stack for Parent Check on AWS:

```
Internet
  → ALB (:80)                       aws_lb + listener + target group
  → ECS Fargate service             aws_ecs_service (task: gunicorn on :8000)
      ├─ RDS PostgreSQL 16 (pgvector)   aws_db_instance
      ├─ ElastiCache Redis 7            aws_elasticache_cluster
      └─ Secrets Manager                SECRET_KEY / DATABASE_URL / REDIS_URL / GEMINI_API_KEY
  + ECR repository                   image registry (deploy.yml pushes here)
  + IAM                              task roles + GitHub OIDC deploy role
```

Security groups chain the tiers: only the ALB reaches the app, only the app
reaches Postgres/Redis. Credentials are generated by Terraform, stored in
Secrets Manager, and injected into the task as env vars — never written to the
task definition or to CI.

## Usage — on-demand up / down (recommended)

This stack (ALB + RDS + ElastiCache + Fargate) costs ~$50/mo if left running.
For a portfolio demo it's cheaper to bring it **up before a demo and destroy it
after** — the whole stack is reproducible from code, so idle cost is ~$0. The
always-on public demo lives on Render; this is the "spin up the real AWS
production stack on demand" story.

**Bring it up (~15 min, RDS is the slow part):**

```bash
cd infra
terraform init
terraform apply -var="gemini_api_key=..."     # creates everything; ECR starts empty

# The ECS service can't start until an image exists in ECR. Build & push, then
# roll the service (from the repo root, Docker running):
aws ecr get-login-password --region eu-west-1 | docker login --username AWS --password-stdin "$(terraform -chdir=infra output -raw ecr_repository_url | cut -d/ -f1)"
docker build -t "$(terraform -chdir=infra output -raw ecr_repository_url):latest" .
docker push "$(terraform -chdir=infra output -raw ecr_repository_url):latest"
aws ecs update-service --cluster parent-check --service parent-check --force-new-deployment --region eu-west-1

terraform output app_url                        # http://<alb-dns> — open once tasks are healthy (~2-3 min)
```

**Tear it down (back to ~$0):**

```bash
terraform destroy -var="gemini_api_key=..."
```

`force_delete` on the ECR repo, `skip_final_snapshot` on RDS, and
`recovery_window_in_days = 0` on the secret are all set so `destroy` completes
cleanly without leftovers that keep billing.

### Optional: enable CI/CD (GitHub Actions deploys on push)

Off by default (`enable_cicd = false`) so the first apply never depends on an
OIDC provider pre-existing. To turn it on:

```bash
terraform apply -var="enable_cicd=true" -var="gemini_api_key=..."
terraform output github_deploy_role_arn    # -> ROLE_ARN in .github/workflows/deploy.yml
terraform output ecr_repository_url        # -> ECR_REPO in deploy.yml
```

This creates the GitHub OIDC provider + a scoped deploy role. If your account
already has the provider, `terraform import` it instead (a second create errors).

## Notes / production hardening

- Uses the **default VPC** for brevity. For production, create a dedicated VPC
  with private subnets for RDS/Redis and keep only the ALB public.
- **HTTP only** (`:80`). Fine for a screen-shared demo on the ALB DNS name; for a
  public always-on URL add an ACM cert + a `:443` listener and redirect `:80 → :443`
  (needs a domain).
- `skip_final_snapshot` and `recovery_window_in_days = 0` are demo/on-demand
  conveniences — remove them if this ever holds real data you can't lose.
